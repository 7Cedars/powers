# AdoptMandates

AdoptMandates is an executive mandate that enables the adoption of new mandates into the Powers protocol. It provides a mechanism to add new governance rules and functionality to the system.

## Overview

This mandate provides a mechanism to:
- Adopt new mandates into the protocol
- Configure mandate parameters
- Set up mandate conditions
- Track mandate adoption history
- Enforce adoption rules

## Configuration

When adopting an AdoptMandates instance, no parameters are required. The mandate is designed to facilitate the adoption of other mandates.

## Usage

### Proposing an Action

When calling the mandate, four parameters must be provided:

1. `mandate` (address): The address of the mandate contract to adopt
2. `nameDescription` (string): Name and description of the mandate
3. `inputParams` (bytes): Encoded input parameters for the mandate
4. `conditions` (bytes): Encoded conditions for the mandate

### Execution Flow

1. **Mandate Validation**
   - Verifies the mandate contract address
   - Validates mandate parameters
   - Checks adoption conditions

2. **Mandate Adoption**
   - Adopts the new mandate
   - Configures mandate parameters
   - Sets up mandate conditions

3. **State Management**
   - Updates adoption history
   - Maintains mandate registry
   - Tracks mandate status

## Technical Specifications

### State Variables

```solidity
struct Data {
    address[] adoptedMandates;     // List of adopted mandates
    mapping(address => bool) isAdopted; // Adoption status
    mapping(address => uint256) adoptionTime; // When mandate was adopted
}

mapping(bytes32 mandateHash => Data) public data;
```

### Functions

#### `initializeMandate`
```solidity
function initializeMandate(
    uint16 index,
    string memory nameDescription,
    bytes memory inputParams,
    Conditions memory conditions,
    bytes memory config
) public override
```
- Initializes mandate with configuration parameters
- Sets up adoption tracking
- Initializes empty mandate registry

#### `handleRequest`
```solidity
function handleRequest(
    address caller,
    address powers,
    uint16 mandateId,
    bytes memory mandateCalldata,
    uint256 nonce
) public view virtual override returns (
    uint256 actionId,
    address[] memory targets,
    uint256[] memory values,
    bytes[] memory calldatas,
    bytes memory stateChange
)
```
- Processes the adoption request
- Validates mandate and parameters
- Prepares adoption data
- Returns execution data

### Error Conditions

1. **Adoption Errors**
   - "Invalid mandate"
   - "Mandate already adopted"
   - "Invalid parameters"

2. **Parameter Errors**
   - "Invalid name"
   - "Invalid conditions"
   - "Invalid input parameters"

3. **Validation Errors**
   - Zero address mandate
   - Invalid mandate contract
   - Invalid adoption status
